用 hexo + github 构建自己的个人博客

发表于 2018-07-11 | 分类于 个人项目

在构建我的这个博客的时候，我踩了无数的坑，花了差不多三天的时候才构建完成，这里分享一下我的建站经历，帮助同样遇到问题的童鞋避坑。

我的系统环境： deepin 15.5 ，一开始我是使用 Win10 和 Ubuntu 16.04 的系统尝试建站，但遇到了各种各样奇奇怪怪的问题，最后使用 deepin 成功建站，所以我的攻略比较适用于使用 deepin 系统的童鞋。

先介绍一下 hexo ，Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

配置Github

首先注册、登录 https://github.com/

记住自己的Username（很重要）

然后右上角选择 Create a new repository

Repository name （填自己的名字） yourname.github.io (yourname与你的注册用户名一致,这个就是你博客的域名了)

例如，我的域名是github.com/fukawachan，就填入fukawachan.github.io

环境安装（node、git）

1、安装 Node.js https://nodejs.org/en/

2、安装 Git https://github.com/waylau/git-for-win

Git教程可以参考廖雪峰老师的教程，廖老师介绍地非常深入浅出，适合初学者学习。

对于使用linux系统的童鞋来说，一般系统内置了Git，可以不用安装。

3、安装Hexo

必备的环境安装好后，使用npm命令即可安装Hexo

sudo npm install -g hexo-cli

由于国内的网速较慢，建议使用cnpm代替npm

sudo npm install -g cnpm --registry=https://registry.npm.taobao.org

安装完成后，使用下列命令查看是否安装成功

cnpm -v

创建博客

在/home中创建一个文件夹，名称任意，比如hexo。然后进入这个文件夹，右键点开终端，输入命令：

hexo init

稍等片刻，速度有点慢。成功提示如下：

INFO  Start blogging with Hexo!

初始化 hexo 之后，source目录下自带一篇hello world文章, 所以直接执行下方命令

hexo g
#启动本地服务器
hexo s
#在浏览器输入 localhost:4000/ 就可以看到网站的模样

现在我们已经可以在本地看到网站初步的样子，接下来我们就是发布到github上。

打开终端，输入：

sh-keygen -t rsa -C "Github的注册邮箱地址"

接下来一路Enter，得到信息：

Your public key has been saved in /home/hexo/.ssh/id_rsa.pub

找到该文件，打开（sublime text），Ctrl + a 复制里面的所有内容，然后在GitHub中打开：https://github.com/settings/ssh

New SSH key ——Title：blog —— Key：输入刚才复制的—— Add SSH key

配置博客

在blog目录下，打开_config.yml文件，修改参数信息

title: 博客名称
subtitle: 副标题
description: 网页描述
author: 你的名字
language: 网站语言
timezone: 时区

注意，这里冒号（：）后面需要空一格才能生效。

配置部署：

deploy: 
  type: git
  repo: https://github.com/yourname/yourname.github.io.git
  branch: master

那么现在，登录 https://www.yourname.github.io ，看看你的网站有没有发布？

发表博文

在博客文件夹的根目录下打开终端，输入如下的命令就可以发表博文了：

hexo n “博文名称” == hexo new “博文名称” #新建文章

hexo g == hexo generate #生成

hexo s == hexo server #启动服务预览

hexo d == hexo deploy #输入账号密码，部署到服务器上

优化博客

Hexo 有一个优点，就是它的模板和第三方插件很多。这里推荐我使用的模板：NexT

安装好模板后，可以进一步对网站的功能进行添加和优化，可以参照这篇博文：hexo的next主题个性化配置教程,因为这位博主已经讲的非常详细，这里就不展开来说了,有兴趣的同学可以通过链接进一步深入优化。
