tensorflow学习笔记0：前言

发表于 2018-07-09 | 分类于 深度学习

点进我的博客的同学，应该都是想要深入学习机器学习领域的，在这里我仅对我自己的学习情况做一个分享。

首先先做个自我介绍，我是一个心理学专业的文科生。数学基础好吗？自我感觉挺差的，没有学数学的那种灵感，考高数是靠刷题拿的高分。

那我学了哪些和数学有关的课程？

大一学了微积分和应用统计。微积分学的非常浅薄，连泰勒展开都没有接触，仅仅是学习了一点简单的微分和积分方法。应用统计学的也很简单，推断统计那一块就学了t检验、方差分析、卡方检验和一元回归，连多元回归都没有学到，更别说聚类这些更深入的内容。

后来到了大二，除了心理测量之外，基本上就没有跟数学有关的课程了。那个时候我查询了一下国内顶尖心理学专业高校的课程，发现和这些学校的学生相比，我自己的数学基础实在是太差了，于是开始了我的自学之旅。

大二上学期重新温习了一遍微积分，然后花了三个月的时间学习线性代数。我看的教材是 David C.Lay 教授的《线性代数及其应用》，不得不说，这本教材真的是相当的适合初学者。David教授把每一个步骤，每一个推算过程都写的非常详细，并且书中图文并茂，把线性关系解释得非常透彻，对于我这种基础差的人看起来真是一种享受。

看完线性代数之后，那个时候有一个同学用Python编了一个贪吃蛇的游戏。对于我这种完全没有编程基础的人来说，看到一行行代码能转变成一个游戏，感到非常神奇，好奇之下，我就入了Python的坑，先学了python2，看的教材是《Hard Way Learn Python》，这本书前半段比较易懂，但到了后半段面向对象编程开始就有点云里雾里了。这个时候听说python2快被淘汰了（雾），于是开始学习python3.这次我看的教程是廖雪峰老师的python教程，廖老师的教程胜在易懂，但后面的习题较难，好几次做到怀疑人生。差不多也是花了三个月时间用课余时间才过完一遍，主要是了解了python的基本语法，面向对象编程和面向函数编程的思想，明白了编程不仅仅只是码代码，更需要规划、构建和数学思维。

然后那个时候收到消息说，还有两个月就要开始数学建模竞赛。学习了这么久的数学知识，也想通过这么一场竞赛来实践检验一下，于是我就报名参赛。这两个月的时间里，我在中国大学MOOC平台上看了用Python玩转数据，这门课深入浅出，介绍了python科学计算的几个重要的包以及它们的用法，包括Scipy、numpy、matplotlib等等。除此之外还有走近数学——数学建模篇，这门课内容比较简单，从人口的增长率、卫星的定位这些和生活有关的内容出发，讲解了数学建模的思想，适合没有基础的同学入门学习。

到了比赛的时候，我的比赛题目是排课问题，查了资料之后，比赛前两天时间快速学了遗传算法，但是我在算法方面的基础太过薄弱，虽然理解了遗传算法的思路，但实在写不出代码，就砍掉了交换的过程，只保留了算法中变异的过程，最后也只拿了一个参与奖。

到现在为止，我掌握的知识主要包括基础微积分、基本的统计基础、线性代数的基础知识、python的编程基础，现在学习了一部分tensorflow的编程方法，目前感觉难度不大。如果你也是一个文科生，害怕编程和数学，那么我的经历也许能给你一点借鉴：学习编程和数学没有想象的那么难。

在开始接下来的内容之前，你可以先把我先前说到的那些基础知识打牢，那么，接下来的知识对你来说就问题不大。

那么让我们开始吧。
